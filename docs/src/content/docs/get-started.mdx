Welcome to the documentation for the Rivu, a modern RSS 2.0 compliant feed
generator for Node.js with type-safety. This guide will help you install the
library, understand basic usage and get up and running in just a few minutes
using the package.

**TIP**: If this is your first time using Rivu, it is recommended to read this
page from start-to-finish.

## Installing Rivu

To install Rivu, you will [Node.js](https://nodejs.org) installed with a package
manager like `npm` accessible on your system. Regardless, to install Rivu with a
package manager of your choice, follow these commands:

<Tabs>
  <TabItem label="npm">
    <Code code={`npm install rivu`} lang="console" />
  </TabItem>
  <TabItem label="yarn">
    <Code code={`yarn install rivu`} lang="console" />
  </TabItem>
  <TabItem label="pnpm">
    <Code code={`pnpm install rivu`} lang="console" />
  </TabItem>
</Tabs>

## Basic Usage

With Rivu installed and accessible, you can get started using the package right
away and generate an RSS XML feed for yourself. The following instructions will
help you generate the feed as a string output in your terminal.

## Contribute to the Project

Contributions to the project are welcome! Here's how you can contribute to the
project in numerous ways:

1. [Report bugs](https://github.com/Jarmos-san/rivu/issues/choose/new) in the
   project and also engage in community discussions and answer questions from
   other users.

2. [Share PRs](https://github.com/Jarmos-san/pulls) with patches and bug fixes
   in the project. Feature enhancements which do not deviate the contextual use

3. [Sponsor the project](https://github.com/sponsor/Jarmos-san) a minimum of $5
   a month to keep the lights on.

To learn more about the ways you can contribute to the project, check out the
[Contributing Guidelines](/contributing-guidelines) document.

## Get Support

Found a bug or need help/clarification on usage? Head over to the project's
repository to open
[a new discussion thread](https://github.com/Jarmos-san/rivu/discussions/new/choose)
or file [a bug report](https://github.com/Jarmos-san/rivu/issues/new/choose)
with your findings. It is recommended to search for existing threads before
creating a new one though. There will always be a good chance, someone else may
already have stumbled across the issue you are facing and may have provided a
solution to it as well.
